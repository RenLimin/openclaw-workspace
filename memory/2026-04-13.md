- **已尝试 25+ 种方案**: headless/有头浏览器、不同 SSO 端点、Cookie 注入、表单提交、JS 调用等
- **根本原因**: IAM-OA SSO 集成未正确配置

  - OA SSO 端点 (`/api/sso/login`, `/login/VerifyLoginWeaversso.jsp`) 返回 "登录信息超时" 或 "非法登录方式"
  - 两个系统使用独立会话，无法共享 Cookie
- **Rex 提示**: OA 首次登录须通过 IAM 页面，点击应用卡片跳转 (有头模式)
- **Cookie 导出**: 已导出到 `training-reports/ella/iam-cookies.json`，但 Cookie 无法建立 OA 会话
- **状态**: ✅ OA 登录已解决!

### ✅ 成功方案 (2026-04-13 16:45)

**OA 登录完整流程**:
1. 登录 IAM (https://iam.bangcle.com/)
2. 调用 API: `GET /api/application/apps/jumpSystem?appid=9`
   - Header: `x-access-token: <token>` (从 localStorage.GlobalState.token 获取)
3. 返回跳转 URL: `https://oa.bangcle.com/api/bangclesso/callback?code=xxx`
4. 访问该 URL → 自动跳转到 OA 主页

**关键发现**:
- Token 在 `x-access-token` header 中，不是 `Authorization: Bearer`
- OA 应用 ID = 9
- 应用列表 API: `/api/application/apps/getPermissionApps`
