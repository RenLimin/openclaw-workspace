# 错误升级规则 v1.0

> **目的**: 定义 Agent 任务失败时的处理流程和上报机制

---

## 一、错误升级链路

```
Agent 任务失败
   │
   ▼
┌─────────────────────────┐
│ 第 1 层: Agent 自处理    │
│ - 自动重试 (最多 3 次)   │
│ - 尝试替代方案           │
│ - 记录错误到 memory      │
└─────────────────────────┘
   │ 3 次失败
   ▼
┌─────────────────────────┐
│ 第 2 层: 上报 Jerry      │
│ - 发送错误详情           │
│ - Jerry 分析并重新分配   │
│ - Jerry 尝试替代方案     │
└─────────────────────────┘
   │ Jerry 3 次失败
   ▼
┌─────────────────────────┐
│ 第 3 层: 上报 Rex        │
│ - 发送完整错误报告       │
│ - 包含已尝试的方案       │
│ - 等待 Rex 决策          │
└─────────────────────────┘
```

---

## 二、错误分类

| 类型 | 示例 | 处理策略 |
|------|------|---------|
| API 超时 | OA/ONES 无响应 | 重试 3 次 → 换网络 → 上报 |
| 认证失败 | Token 过期 | 刷新 Token → 重试 → 上报 |
| 权限不足 | 无法访问资源 | 直接上报 Rex |
| 依赖缺失 | 技能/库未安装 | 尝试安装 → 失败上报 |
| 数据错误 | 返回格式不对 | 记录样本 → 上报 |

---

## 三、错误报告格式

```markdown
# 错误报告 - {Agent} - {时间}

## 错误信息
- **Agent**: {name}
- **任务**: {task description}
- **错误**: {error message}
- **时间**: {timestamp}

## 已尝试方案
1. {方案 1} - 结果: {成功/失败}
2. {方案 2} - 结果: {成功/失败}
3. {方案 3} - 结果: {成功/失败}

## 紧急度
- P0: 需要立即处理
- P1: 24h 内处理
- P2: 可延后处理
```
