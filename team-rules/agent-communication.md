# 跨 Agent 通信规则 v1.0

> **目的**: 定义 Agent 之间的任务传递和协作规范
> **原则**: 通过 Jerry 协调，不直接互相调用

---

## 一、通信架构

```
┌─────────────────────────────────────────┐
│                Jerry 🦞                  │
│           (任务协调中心)                  │
│                                         │
│  接收 Rex 指令 → 分析意图 → 分发任务     │
│  收集 Agent 结果 → 汇总 → 汇报 Rex       │
└─────────────┬───────────────────────────┘
              │
     ┌────────┼────────┬────────┐
     │        │        │        │
  Ella 🦊  Oliver 🐘 Aaron 🦉 Iris 🐦‍⬛
```

---

## 二、通信方式

### 2.1 任务分发

Jerry 通过 `sessions_spawn` 向 Agent 分发任务：

```python
sessions_spawn(
    agentId="ella",
    task="请审核这份合同：{合同内容}",
    mode="run"
)
```

### 2.2 结果返回

Agent 完成后返回结果给 Jerry：

```
Agent 执行任务 → 生成结果报告 → 返回给 Jerry → Jerry 汇总 → 汇报 Rex
```

### 2.3 异常通知

Agent 遇到无法解决的问题时：

```
Agent 发现问题 → 记录到 memory → 通知 Jerry → Jerry 决定处理方式
```

---

## 三、协作场景

| 场景 | 流程 | 说明 |
|------|------|------|
| 合同 + 项目 | Ella → Jerry → Oliver | 合同通过后创建项目 |
| 项目 + 经营 | Oliver → Jerry → Aaron | 项目进度同步用于经营分析 |
| 邮件 + 合同 | Iris → Jerry → Ella | 合同邮件转发审批 |

---

## 四、注意事项

1. **不直接调用**: Agent 间不直接调用，必须通过 Jerry 协调
2. **记录所有通信**: 每次任务分发和结果都记录到 memory
3. **超时处理**: 任务超过预期时间未完成，Jerry 主动跟进
4. **权限隔离**: 每个 Agent 只能访问自己的工作区
