# 🎯 智能体团队 — 训练规则 v2.0

**生效日期**: 2026-04-11
**适用范围**: 全部 5 个智能体

---

## 1. 训练前必读

每次训练开始前，**必须**按顺序读取：

1. `team-rules/training-rules.md`（本文件）
2. `agents/{name}/memory/training-memory.md`（本 agent 训练记忆）
3. `team-rules/general-rules.md`（通用规则，确认角色和通信协议）
4. `team-rules/security-rules.md`（安全规则，确认边界）

## 2. 训练记忆本机制

### 2.1 位置

agents/{name}/memory/training-memory.md

### 2.3 更新规则
- 每完成一个训练阶段**必须**更新
- 遇到新问题**必须**记录
- 做出关键决策**必须**记录原因
- 成功经验更新到 FAQ

## 3. 并行训练规则

### 3.1 隔离性
- 各 agent 有独立的 `sessions/`、`workspace/`、`memory/`
- 并行训练互不干扰
- 每个 agent 的训练进度独立记录

### 3.2 训练分派
- Jerry 负责任务分解和分派
- 每个 agent 接到训练任务后独立执行
- 训练完成后向 Jerry 汇报进度

### 3.3 训练冲突处理
- 如果多个 agent 需要训练同一共享技能，Jerry 负责协调顺序
- 如果训练结果影响其他 agent，Jerry 负责同步

## 4. 训练阶段定义

### Phase 1: 环境搭建
- 目录结构创建
- 配置文件初始化
- 基础工具安装

### Phase 2: 技能掌握
- 读取技能文档
- 执行练习任务
- 验证功能正常

### Phase 3: 角色训练
- 理解自身职责边界
- 练习通信协议
- 端到端测试

### Phase 4: 协作演练
- 与其他 agent 协作测试
- 异常场景处理
- 汇报流程验证

## 5. 训练成果验收

- ✅ 训练记忆本已更新
- ✅ 专属技能可正常使用
- ✅ 能正确接收 Jerry 的任务分派
- ✅ 能正确向 Jerry 汇报结果
- ✅ 遵守安全规则
- ✅ 代码/文件已提交 Git
