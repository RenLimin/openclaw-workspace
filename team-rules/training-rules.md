# 🎯 智能体团队 — 训练规则 v2.0

**生效日期**: 2026-04-11
**适用范围**: 全部 5 个智能体

---

## 1. 训练前必读

每次训练开始前，**必须**严格按以下顺序读取：

1. `team-rules/security-rules.md` ← 安全第一，确认边界
2. `team-rules/general-rules.md` ← 确认角色和通信协议
3. `team-rules/training-rules.md` ← 确认训练流程
4. `agents/{name}/memory/training-memory.md` ← 回顾历史，从零开始

## 3. 并行训练规则

### 3.1 隔离性
- 各 agent 有独立的 `sessions/`、`workspace/`、`memory/`
- 并行训练互不干扰
- 每个 agent 的训练进度独立记录

### 3.2 训练分派
- Jerry 负责任务分解和分派
- 每个 agent 接到训练任务后独立执行
- 训练完成后向 Jerry 汇报进度

### 3.3 训练冲突处理
- 如果多个 agent 需要训练同一共享技能，Jerry 负责协调顺序
- 如果训练结果影响其他 agent，Jerry 负责同步

## 4. 训练阶段定义

### Phase 1: 环境搭建
- 目录结构创建
- 配置文件初始化
- 基础工具安装

### Phase 2: 技能掌握
- 读取技能文档
- 执行练习任务
- 验证功能正常

### Phase 3: 角色训练
- 理解自身职责边界
- 练习通信协议
- 端到端测试

### Phase 4: 协作演练
- 与其他 agent 协作测试
- 异常场景处理
- 汇报流程验证

### Phase 3: 角色训练 — 验收标准
- ✅ 能准确理解自身职责边界，不越权操作
- ✅ 能正确解读 Jerry 的任务分派，明确 P0/P1/P2 含义
- ✅ 能按通信格式规范汇报结果
- ✅ 遇到超出职责范围的任务，能识别并上报 Jerry

### Phase 4: 协作演练 — 验收标准
- ✅ 能通过 Jerry 协调完成跨 agent 任务
- ✅ 能正确处理任务依赖关系（如：合同审批需要项目信息）
- ✅ 异常场景不崩溃，按错误处理机制执行
- ✅ 能识别 P0 紧急任务并立即响应

## 5. 训练失败处理

### 5.1 重试机制
- Agent 训练卡住时，自动重试 3 次（间隔递增）
- 3 次重试后仍卡住 → 标记为"训练阻塞"，记录原因，汇报给 Jerry

### 5.2 Jerry 介入
- Jerry 收到训练阻塞报告后，分析原因
- Jerry 尝试 3 种不同方案帮助该 agent 突破
- 3 种方案均无效 → 上报 Rex，请求人工干预

### 5.3 跳过策略
- 如果某个训练步骤非阻塞核心功能，经 Jerry 确认后可跳过
- 跳过的步骤必须记录到 training-memory.md，标注"已跳过 - 原因"

- ✅ 专属技能可正常使用
- ✅ 能正确接收 Jerry 的任务分派
- ✅ 能正确向 Jerry 汇报结果
- ✅ 遵守安全规则
- ✅ 代码/文件已提交 Git
